  • Activity:

    2. Buksan ang folder na 'yan sa VS Code (File > Open Folder...).
    3. Gumawa ng bagong file, pangalanan itong index.html.

‚è≥ PHASE 2: Your Time-Traveling Journal - Git! ~40 Mins~

Imagine Git as your **Time-Traveling Journal**. Bawat importanteng ginawa mo, ililista mo, at pwede kang bumalik sa nakaraan kung sakaling magkamali!

2.1. Why a Journal? The Power of Git (10 mins)

Bakit kailangan ito? Sino ba naman ang may ayaw sa superpower na ito?

  • **"Oh no, I broke it!"** Kapag nagkamali ka, pwede kang bumalik sa previous version ng code mo.
  • **"What did I change?"** Nakikita mo kung ano ang mga binago mo sa bawat entry.
  • **"Collaboration!"** Makakatrabaho mo ang ibang developers sa iisang project nang hindi nagkakagulo.
Analogy: Parang naglalaro ka ng RPG. Bawat "commit" mo, parang "save point"!

2.2. The Three Secret States of Your Code! (15 mins)

Para makagawa ng journal entry (commit), dadaan muna ang code mo sa tatlong secret states:

‚úèÔ∏è

MODIFIED

Status: May binago ka sa code mo sa VS Code, pero hindi pa ito handa para sa "snapshot."

Analogy: Nagsusulat ka sa draft ng journal, pero hindi mo pa fina-finalize.

üì¶

STAGED (INDEX)

Action: git add .

Status: Pinili mo na kung aling changes ang isasama sa susunod na journal entry. Naka-pack na ang box!

‚úÖ

COMMITTED

Action: git commit -m "Your Message"

Status: Success! Na-save na ang journal entry sa **local** Time-Traveling Journal mo!

2.3. Hands-on Journaling: Git Commands (15 mins)

Let's try it! Gamitin natin ang Terminal sa VS Code para mag-journal.

  • Activity:
    1. Initialize ang Journal: Sa VS Code Terminal, i-type: git init
      Output: Initialized empty Git repository in ...MeganDevQuest/.git/

      Ibig sabihin, handa na ang folder mo para maging Git repository!

    2. Modified Stage: Gumawa ka ng change. Halimbawa, palitan ang <p> tag mo: <p>This is my first awesome webpage, now with super colors!</p>
    3. Staged Stage: Ilagay sa "Staging Area" ang change mo. git add .

      Ang . means "all changes in this folder."

    4. Committed Stage: Isulat na sa journal! git commit -m "Added my first awesome webpage content"
      Output: [main (root-commit) ...] Added my first awesome webpage content
      1 file changed, 7 insertions(...)

      Congrats! May first entry ka na sa time machine mo!

CHALLENGE! (Practice makes a Code Hero!)
Magdagdag ng <h2>My Superpowers:</h2> sa index.html mo. Pagkatapos, ulitin ang git add . at git commit -m "Added my superpowers section".

üåê PHASE 3: Your Global Archive - GitHub! ~30 Mins~

Ang GitHub ang iyong **Global Archive** sa internet! Dito mo isasave ang Time-Traveling Journal mo para **safe** (kahit masira ang laptop mo!) at **shareable** sa buong mundo!

3.1. Setting up Your GitHub Account & Repository (10 mins)

Para makapag-upload sa Global Archive, kailangan mo ng account at isang "vault" (repository).

  • Activity:
    1. Kung wala pa, gumawa ng account sa github.com.
    2. Gumawa ng bagong repository. Click "New" button.
    3. Pangalanan itong megans-first-quest. Gawing **Public**. Don't add a README yet.
    4. Pagkatapos, makikita mo ang mga commands para i-connect ang local project mo.

3.2. Linking Your Journal to the Archive (10 mins)

I-connect na natin ang local journal mo sa online archive!

  • Activity:
    1. Kopyahin ang command na mukhang ito (usually nasa ilalim ng "‚Ä¶or push an existing repository from the command line"): git remote add origin https://github.com/your-username/megans-first-quest.git
      git branch -M main
      git push -u origin main
    2. I-paste at i-run ang mga commands na 'yan sa VS Code Terminal mo.

Resulta: Pagkatapos mag-run, i-refresh ang GitHub page mo. *Boom!* Makikita mo na ang files mo online!

**Fun Fact:** Ang origin ay ang common name ng "main" remote repository mo. Ang main naman ang pangalan ng default branch mo.

3.3. The Full Pro Flow: Edit, Add, Commit, Push! (10 mins)

Eto na ang buong cycle na ginagawa ng mga Code Heroes sa araw-araw!

  • Activity:
    1. EDIT: Sa index.html, magdagdag ng isa pang <p> tag: <p>I just pushed my code to GitHub! How cool is that?!</p>
    2. ADD: Ilagay sa Staging Area: git add .
    3. COMMIT: Isulat sa journal: git commit -m "Added a paragraph after pushing to GitHub"
    4. PUSH: I-upload sa Global Archive: git push origin main

Check: Balikan ang GitHub page mo. Makikita mo na ang latest changes mo, pati na ang bagong commit message!
